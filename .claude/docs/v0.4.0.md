# v0.4.0 — CLI 增强

## 目标

四项增强：新增 `remove` / `rename` 命令，优化 `templatize` 来源与存储交互，重设计 `dev` / `build` / `start` 命令为交互式。

---

## 一、共用工具函数：listUserProjects()

多个命令都需要「列出用户创建的项目」，提取为公共逻辑，放在 `scripts/workspace.mjs`。

**定义**：读取 `pnpm-workspace.yaml` 的 `packages` 列表，排除以下条目：

- `packages/*`
- `templates/*`
- glob 模式（含 `*` 的条目）

返回实际存在于根目录的目录名数组（即用户创建的项目）。

## 二、remove 命令（`scripts/remove.mjs`）

### 交互流程

```
$ pnpm remove

? 选择要删除的项目（多选）
  ◯ web-app
  ◯ my-app

? 确认删除以下项目？web-app, my-app (y/N)

✓ 已删除 web-app/
✓ 已删除 my-app/
✓ 已更新 pnpm-workspace.yaml
```

- 使用 `@inquirer/prompts` 的 `checkbox` 做多选
- 二次确认使用 `confirm`，默认 N；取消则退出不做任何操作
- 删除：`fs.rmSync(dir, { recursive: true })`
- 更新 workspace：移除对应条目

### 边界处理

- 无用户项目时报错退出：`错误：当前 workspace 没有用户创建的项目`

## 三、rename 命令（`scripts/rename.mjs`）

### 交互流程

```
$ pnpm rename

? 选择要重命名的项目
  ❯ web-app
    my-app

? 新名称 (web-app): new-name

✓ 已重命名目录 web-app/ → new-name/
✓ 已更新 package.json name 字段
✓ 已更新 pnpm-workspace.yaml
```

- 单选 `select`
- 输入新名称，默认值为当前名称
- 若新名称与当前相同，报错退出：`错误：新名称与当前名称相同`
- 若目标目录已存在，报错退出：`错误：new-name/ 已存在`

### 操作

1. `fs.renameSync(oldPath, newPath)` 重命名目录
2. 若项目含 `package.json`，更新其 `name` 字段为新名称
3. 更新 `pnpm-workspace.yaml`：将旧条目替换为新条目

## 四、templatize 优化（更新 `scripts/templatize.mjs`）

### 变更点

**来源选择**：从硬编码 `['frontend', 'backend']` 改为 `listUserProjects()`

**存储位置**：

旧：存到 `templates/<sourceType>/<templateName>`（sourceType 固定为 `frontend` / `backend`）

新：两步选择

1. 选择模板类型目录（列出 `templates/` 下已有的子目录，附加「+ 新建类型」选项）
2. 在选定类型下选择已有模板（附加「+ 新建模板」选项）或直接新建

### 新交互流程

```
$ pnpm templatize

? 选择要模板化的项目
  ❯ web-app

? 选择模板类型
  ❯ frontend
    backend
    + 新建类型

（若选「+ 新建类型」）
? 类型名称: electron

? 选择模板
  ❯ nextjs-basic  （更新）
    + 新建模板

（若选「+ 新建模板」）
? 模板名称: nextjs-tailwind

✓ 已保存为 templates/frontend/nextjs-tailwind
```

- 若 `templates/` 不存在或为空，第一步直接输入类型名称
- `package.json` 的 `name` 字段更新为模板名称（同旧逻辑）

## 五、dev / build / start 重设计

### 变更概述

- 移除 `package.json` 中的 `dev:fe`、`dev:be`、`build:fe`、`build:be`、`start:fe`、`start:be`
- `dev` / `build` / `start` 三个脚本改为调用对应的 `.mjs` 文件

### 新增脚本（`scripts/run.mjs`，接收命令参数）

通过 `process.argv[2]` 接收 `dev` / `build` / `start`，三个 pnpm 脚本共用一个文件：

```
"dev":   "node scripts/run.mjs dev",
"build": "node scripts/run.mjs build",
"start": "node scripts/run.mjs start"
```

### 交互流程

```
$ pnpm dev

? 选择要运行的项目
  ❯ 全部（并行）
    web-app
    my-app

✓ 运行 web-app dev ...
✓ 运行 my-app dev ...
```

- 使用 `select` 单选
- 选「全部（并行）」：对所有用户项目执行 `pnpm --filter <name> run <cmd>`，通过 `child_process.spawn` 并行启动，继承 stdio
- 选具体项目：仅执行该项目的命令

### 边界处理

- 无用户项目时报错退出：`错误：当前 workspace 没有用户创建的项目`
- 若只有一个项目，跳过选择步骤直接运行
