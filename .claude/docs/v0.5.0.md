# v0.5.0 — contribute 命令

## 目标

新增 `pnpm contribute` 命令，允许从子项目将 `packages/` 或 `templates/` 的改动贡献回
`fullstack-starter` 的 `cli-release` 分支，并自动创建 PR。

---

## 场景

```
fullstack-starter (cli-release/main)   ← 目标：往这里发 PR
        ↑ clone
新项目（有自己的 git remote）
  ├─ 开发了新的 packages/my-pkg
  └─ 想把这些改动贡献回 fullstack-starter
```

---

## contribute 命令（`scripts/contribute.mjs`）

### 交互流程

```
$ pnpm contribute

使用已记录的 starter 地址：git@github.com:user/fullstack-starter.git
正在 fetch upstream...

? 选择要贡献的内容（多选）
  ◯ packages/ui
  ◯ packages/hooks
  ◯ packages/utils
  ◯ templates/frontend/nextjs-basic

? 贡献分支名称 (contrib/packages-ui): my-feature

? Commit 信息 (contrib: packages/ui): feat: add dark mode support

✓ 已复制 packages/ui
✓ 已推送到 upstream/my-feature

? PR 标题 (feat: add dark mode support): feat(ui): add dark mode support

（自动打开 PR 创建页面或显示链接）
```

### 首次运行（无 `.starter-remote`）

```
$ pnpm contribute

? Starter 仓库地址（git remote URL）: git@github.com:user/fullstack-starter.git
✓ 已记录到 .starter-remote
```

---

## 实现逻辑

### 1. 读取 / 记录 starter 地址

- 读取 `.starter-remote`（首次 `input` 询问并写入）

### 2. 配置 upstream remote（幂等）

- 若无 `upstream` → `git remote add upstream <url>`
- 若已有但 URL 不同 → `git remote set-url upstream <url>`

### 3. `git fetch upstream cli-release`

### 4. 构建候选列表

- `packages/` 下各子目录
- `templates/<type>/<name>` 两级子目录
- 若两者均不存在 → 报错退出

### 5. `checkbox` 多选要贡献的目录

### 6. 询问分支名称

- 默认：`contrib/<第一个选项名称（/ 替换为 -）>`

### 7. 创建临时 worktree（基于 `upstream/cli-release`）

```bash
git worktree add -b <branchName> <tmpPath> upstream/cli-release
```

### 8. 复制选中目录到 worktree

- 排除：`node_modules`、`.next`、`dist`、`build`、`coverage`、`.turbo`、`.env*`、`.DS_Store`
- 覆盖目标路径已有内容

### 9. 检测变更并提交

- `git add -A` → 若无变更（porcelain 为空）→ 提示后退出
- `input` 询问 commit 信息，默认 `contrib: <选中项列表>`
- `git commit`

### 10. 推送并创建 PR

- `git push upstream <branchName>`
- 从 URL 提取 GitHub `owner/repo`（支持 https 和 ssh 格式）
- 若可提取：`gh pr create --repo <owner/repo> --base cli-release --head <branchName>`
  - `input` 询问 PR 标题（默认同 commit 信息）
- 若不可提取（非 GitHub 或解析失败）：打印手动创建链接

### 11. 清理 worktree

```bash
git worktree remove <tmpPath> --force
git branch -D <branchName>   # 删除本地临时分支
```

## 其他变更

### `.gitignore.main`

追加 `.starter-remote`（每个子项目的 starter 地址各不同，不应提交）

```json
"contribute": "node scripts/contribute.mjs"
```
