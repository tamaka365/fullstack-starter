# v0.1.0 — Monorepo 基础搭建

## 目标

搭建 pnpm monorepo 骨架，为后续模板和脚本开发打好地基。

## 技术决策

- 包消费方式：**source-only**，直接引用 TS 源码，无构建步骤
- 样式方案：`@starter/ui` 配置 vanilla-extract
- 脚本占位：frontend / backend 相关脚本先写占位，等对应目录存在后补全

---

## 步骤

### 步骤 1：根目录 package.json

创建 `/package.json`，包含：

- `name`：`fullstack-starter`，`private: true`
- `engines`：`node >=20`，`pnpm >=9`
- `packageManager`：`pnpm@latest`
- `scripts`：

  | 脚本 | 命令 |
  |------|------|
  | `create` | `node scripts/create.mjs` |
  | `templatize` | `node scripts/templatize.mjs` |
  | `dev` | `pnpm --parallel run dev` |
  | `dev:fe` | `pnpm --filter frontend run dev` |
  | `dev:be` | `pnpm --filter backend run dev` |
  | `start` | `pnpm --parallel run start` |
  | `start:fe` | `pnpm --filter frontend run start` |
  | `start:be` | `pnpm --filter backend run start` |
  | `build` | `pnpm --parallel run build` |
  | `build:fe` | `pnpm --filter frontend run build` |
  | `build:be` | `pnpm --filter backend run build` |
  | `typecheck` | `pnpm --recursive run typecheck` |
  | `prepare` | 现有的 git hooks 配置脚本（保留） |

- `devDependencies`：`@inquirer/prompts`、`typescript`

状态：[x] 已完成

---

### 步骤 2：pnpm-workspace.yaml

创建 `/pnpm-workspace.yaml`，声明：

```yaml
packages:
  - 'packages/*'
  - 'templates/*'
  - 'frontend'
  - 'backend'
```

状态：[x] 已完成

---

### 步骤 3：tsconfig.base.json

创建 `/tsconfig.base.json`，作为所有包的继承基础：

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "lib": ["ES2022"],
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "strict": true,
    "skipLibCheck": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "esModuleInterop": true,
    "isolatedModules": true,
    "noEmit": true
  }
}
```

状态：[x] 已完成

### 步骤 4：@starter/utils 骨架

创建 `packages/utils/`：

**package.json**
- `name`：`@starter/utils`
- `private: true`
- `exports`：`{ ".": "./src/index.ts" }`
- `devDependencies`：`typescript`

**tsconfig.json**
- 继承 `../../tsconfig.base.json`
- `include`：`["src"]`

**src/index.ts**
- 空导出占位：`export {}`
