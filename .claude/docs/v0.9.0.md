# v0.9.0 — Sidebar + SidebarNav 组件

## 目标

在 `packages/ui` 中实现侧边栏布局组件 `Sidebar` 和导航组件 `SidebarNav`，两者通过
`SidebarContext` 共享折叠状态。

- `Sidebar`：持有 `collapsed` 状态，提供 `header` / `footer` slot，中间区域可滚动，
  CSS transition 宽度过渡动画
- `SidebarNav`：基于 `MenuList` 封装，展开态正常显示，折叠态仅 icon + hover Popover
  弹出完整子菜单；`linkComponent` prop 处理路由/外链

## API

### Sidebar

```tsx
<Sidebar
  header={<Logo />}
  footer={<UserInfo />}
  collapsed={collapsed}           // 受控；不传则内部维护
  onCollapsedChange={setCollapsed}
  defaultCollapsed={false}        // 非受控初始值
  expandedWidth={240}             // 展开宽度（px），默认 240
  collapsedWidth={56}             // 折叠宽度（px），默认 56
>
  <SidebarNav items={items} linkComponent={Link} activeKey={pathname} />
</Sidebar>
```

#### SidebarProps

| prop                | 类型                         | 默认值  | 说明                                    |
| ------------------- | ---------------------------- | ------- | --------------------------------------- |
| `children`          | `React.ReactNode`            | —       | 中间可滚动区域内容，通常放 SidebarNav   |
| `header`            | `React.ReactNode`            | —       | 顶部固定区域（logo、品牌名等）          |
| `footer`            | `React.ReactNode`            | —       | 底部固定区域（用户信息、退出等）        |
| `collapsed`         | `boolean`                    | —       | 受控折叠状态                            |
| `onCollapsedChange` | `(collapsed: boolean) => void` | —     | 受控时的状态变更回调                    |
| `defaultCollapsed`  | `boolean`                    | `false` | 非受控初始折叠状态                      |
| `expandedWidth`     | `number`                     | `240`   | 展开宽度（px）                          |
| `collapsedWidth`    | `number`                     | `56`    | 折叠宽度（px）                          |

### SidebarContext

```ts
interface SidebarContextValue {
  collapsed: boolean
  toggle: () => void
}
```

通过 `useSidebar()` hook 消费，供 `SidebarNav` 和调用方使用（如折叠按钮）。

### SidebarNav

```tsx
<SidebarNav
  items={items}
  linkComponent={Link}    // Next.js Link / React Router Link 等
  activeKey={pathname}
/>
```

#### SidebarNavItemData

```ts
interface SidebarNavItemData {
  key: string
  label: string
  icon?: React.ReactNode  // 折叠态必须有 icon 才能显示
  href: string
  children?: SidebarNavItemData[]  // 子菜单（仅叶子节点可 href 跳转）
  disabled?: boolean
}
```

#### SidebarNavProps

| prop            | 类型                                     | 默认值 | 说明                                          |
| --------------- | ---------------------------------------- | ------ | --------------------------------------------- |
| `items`         | `SidebarNavItemData[]`                   | —      | 导航条目数据                                  |
| `linkComponent` | `React.ElementType`                      | `'a'`  | 路由 Link 组件；外链自动降级为原生 `<a>`      |
| `activeKey`     | `string`                                 | —      | 当前激活的条目 key（通常为当前路由 pathname） |

## 行为规格

### Sidebar 宽度动画

- vanilla-extract `style()` 定义 sidebar 容器，宽度由 CSS variable `--sidebar-width` 控制
- 通过内联 style 设置 `--sidebar-width` 值，collapsed 时为 `collapsedWidth`px，否则为 `expandedWidth`px
- CSS `transition: width 0.25s ease` 实现平滑过渡

```ts
// Sidebar.css.ts
export const sidebar = style({
  width: 'var(--sidebar-width)',
  transition: 'width 0.25s ease',
  overflow: 'hidden',  // 折叠时裁剪内容
  display: 'flex',
  flexDirection: 'column',
  height: '100%',
  flexShrink: 0,
})

export const scrollArea = style({
  flex: 1,
  overflowY: 'auto',
  overflowX: 'hidden',
})
```

### SidebarNav 展开/折叠态行为

**展开态**（`collapsed === false`）：

- 渲染完整 MenuList，显示 icon + label
- 有子项的父节点可内联展开/折叠（MenuList 原生行为）
- 所有叶子节点通过 `linkComponent` 渲染为可点击链接

**折叠态**（`collapsed === true`）：

- 仅渲染每个顶层条目的 icon（无 label）
- hover 弹出 Popover，内容为完整 MenuList（含子项展开）
- 没有 icon 的顶层条目：显示 label 首字符作为 fallback

### linkComponent 与外链处理

```tsx
// 内部判断逻辑
const isExternal = href.startsWith('http://') || href.startsWith('https://')
const Comp = isExternal ? 'a' : linkComponent

const extraProps = isExternal
  ? { target: '_blank', rel: 'noopener noreferrer' }
  : {}

return <Comp href={href} {...extraProps}>{children}</Comp>
```

### 受控/非受控折叠状态

与 Popover open 处理方式一致：

```ts
const isControlled = collapsed !== undefined
const [internalCollapsed, setInternalCollapsed] = useState(defaultCollapsed ?? false)
const isCollapsed = isControlled ? collapsed! : internalCollapsed
const toggle = useCallback(() => {
  if (!isControlled) setInternalCollapsed(v => !v)
  onCollapsedChange?.(!isCollapsed)
}, [isControlled, isCollapsed, onCollapsedChange])
```

## 注意事项

- `SidebarNav` 折叠态的 Popover content 中的 MenuList 应与展开态一致，包含完整层级
- `overflow: hidden` 在宽度过渡时会裁剪 label 文字，无需额外处理（width 动画完成前内容自然截断）
- `useSidebar` 在 Sidebar 外部调用会返回 `{ collapsed: false, toggle: noop }`（安全降级），无需强制要求在 Sidebar 内使用
- SidebarNav 的 `renderItem` 传给 MenuList 时，active 样式由 `isActive` 控制，`linkComponent` 只负责渲染容器
