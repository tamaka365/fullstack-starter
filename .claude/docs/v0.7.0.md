## 数据类型（types.ts）

```ts
export interface MenuItemData {
  key: string
  label: string
  icon?: React.ReactNode
  children?: MenuItemData[]
  disabled?: boolean
}

export interface MenuGroupData {
  groupLabel?: string
  items: MenuItemData[]
}

export type MenuListItem = MenuItemData | MenuGroupData

// type guard
export function isMenuGroup(item: MenuListItem): item is MenuGroupData {
  return 'items' in item
}

export interface RenderItemMeta {
  isActive: boolean
  level: number   // 0 = 顶层，1 = 一级嵌套，以此类推
}
```

## Props 设计

```ts
export interface MenuListProps {
  items: MenuListItem[]

  /** 当前激活条目的 key */
  activeKey?: string

  /** 多态渲染：调用方决定条目元素类型与内容 */
  renderItem: (item: MenuItemData, meta: RenderItemMeta) => React.ReactNode

  /** 非受控：初始展开的条目 key 列表 */
  defaultExpandedKeys?: string[]
  /** 受控：当前展开的条目 key 列表 */
  expandedKeys?: string[]
  /** 受控：展开状态变化回调 */
  onExpandedKeysChange?: (keys: string[]) => void
}
```

## 样式（MenuList.css.ts）

- `list`：`list-style: none; margin: 0; padding: 0`，竖向 flex 容器
- `group`：与 `list` 相同，作为分组的子列表容器
- `groupLabel`：分组标题，`font-size` 较小，`color` 弱化，`padding` 留白
- `item`：单个条目容器，`display: flex; align-items: center`
- `itemActive`：激活态覆盖样式（背景色高亮）
- `itemDisabled`：禁用态（`opacity`、`pointer-events: none`、`cursor: not-allowed`）
- `itemIndent`：嵌套缩进，按 `level` 动态设置 `paddingInlineStart`（通过 CSS variable 传入）
- `expandIcon`：展开/收起箭头，`transition` 旋转动画
- `expandIconOpen`：旋转 90deg

## 组件实现要点（MenuList.tsx）

### 展开状态管理

内部用 `useState<Set<string>>` 维护展开 key 集合：
- 非受控：初始值来自 `defaultExpandedKeys`
- 受控：同步外部 `expandedKeys`（转为 Set）
- toggle 时：非受控更新内部 state，受控调用 `onExpandedKeysChange`

### 递归渲染

```
renderItems(items: MenuListItem[], level: number)
  for each item:
    if isMenuGroup → renderGroup(item, level)
    else            → renderItem(item, level)

renderGroup(group, level)
  → <div role="group">
      if groupLabel → <div className={styles.groupLabel}>
      → renderItems(group.items, level)

renderItem(item, level)
  → <div role="none">
      <div
        role="menuitem"
        aria-disabled={disabled}
        tabIndex={disabled ? -1 : 0}
        style={{ '--level': level }}   ← CSS variable for indent
        className={clsx(styles.item, isActive && styles.itemActive, disabled && styles.itemDisabled)}
        onClick={handleClick}
        onKeyDown={handleKeyDown}
      >
        {renderItem(item, { isActive, level })}   ← 调用方渲染
        {hasChildren && <span className={expandIconClass}>▶</span>}
      </div>
      {hasChildren && isExpanded && renderItems(item.children, level + 1)}
```

### 键盘导航（基础）

`onKeyDown` 处理：
- `ArrowDown`：焦点移到下一个可见条目（`focusNextItem()`）
- `ArrowUp`：焦点移到上一个可见条目（`focusPrevItem()`）
- `Enter` / `Space`：触发点击（激活或展开/收起）

可见条目列表通过 `querySelectorAll('[role="menuitem"]:not([aria-disabled="true"])')` 动态获取。

### ARIA

- 顶层容器：`role="menu"`
- 条目：`role="menuitem"`，`aria-disabled`，`aria-expanded`（有子项时）
- 分组：`role="group"`，`aria-label={groupLabel}`
