# v0.11.0 — SidebarNav 折叠态弹出菜单修复

## 概述

修复 `SidebarNav` 折叠态弹出菜单的三个问题：弹出内容错误、缺少背景色、折叠/展开态容器样式不一致。

## 问题与方案

### 1. 弹出内容错误

**现状**：`items={[toMenuItemData(item)]}` 将父节点本身也作为一行渲染，弹出菜单会显示父节点 + 子节点。

**修复**：改为 `items={item.children!.map(toMenuItemData)}`，只展示子节点列表。

**叶子节点**：无子菜单时移除 `Popover`，改用 `<Tooltip placement="right">` 包裹图标按钮，显示 `item.label`。

### 2. 弹出菜单缺背景

**现状**：浮层 `MenuList` 直接渲染，无背景色，与页面内容重叠。

**修复**：在 `SidebarNav.css.ts` 新增 `popoverMenu` class，将 `MenuList` 包裹在 `<div className={styles.popoverMenu}>` 中。

```ts
export const popoverMenu = style({
  background: '#fff',
  borderRadius: 8,
  boxShadow: '0 4px 16px rgba(0,0,0,0.12)',
  padding: '4px 0',
  minWidth: 160,
})
```

### 3. 折叠/展开态样式不一致

**现状**：`nav` 和 `collapsedNav` 各自独立声明 `display: flex; flex-direction: column`，后续若需调整基础样式须同步改两处。

**修复**：提取内部 `navBase` 作为共享 style（不导出），两者通过 vanilla-extract 数组组合复用：

```ts
const navBase = style({
  display: 'flex',
  flexDirection: 'column',
})

export const nav = style([navBase])

export const collapsedNav = style([navBase, {
  alignItems: 'center',
  gap: 2,
}])
```

> `collapsedNav` 的 `padding: '4px 0'` 同步移除，与 `nav` 保持一致，待后续统一加 padding 时再两者���步添加。

## 涉及文件

- `packages/ui/src/sidebar/SidebarNav.css.ts`：提取 `navBase`、新增 `popoverMenu`、删除 `collapsedNav` 的 `padding`
- `packages/ui/src/sidebar/SidebarNav.tsx`：修复弹出内容、叶子节点改用 `Tooltip`、弹出内容包裹 `popoverMenu`
