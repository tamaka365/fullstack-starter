# v0.3.0 — new 命令

## 目标

实现 `scripts/new.mjs`：从 `templates/` 目录选择模板，在根目录创建新项目，并完成依赖去重 + 安装。

## 前提调整

`pnpm-workspace.yaml` 移除默认的 `frontend` / `backend` 条目，改为由 `pnpm new` 在创建项目时动态添加：

```yaml
packages:
  - 'packages/*'
  - 'templates/*'
```

## 交互流程

```
$ pnpm new

? 选择项目类型
  ❯ frontend

? 选择模板
  ❯ nextjs-basic

? 项目名称 (frontend): my-app

✓ 已创建 my-app/
✓ 已更新 pnpm-workspace.yaml
✓ 已从项目 package.json 移除根目录已有的依赖
✓ 依赖安装完成

下一步：
  cd my-app
  pnpm dev
```

- 只显示 `templates/` 下实际存在的类型（`frontend` / `backend`）
- 若只有一种类型，跳过第一步直接选模板
- 若类型下只有一个模板，跳过第二步直接询问名称
- 项目名称默认值：所选类型名（`frontend` 或 `backend`）
- 若目标路径已存在，报错退出（exit code 1）

## 复制规则

从 `templates/<type>/<template>/` 复制到 `<name>/`，忽略以下内容：

| 类型 | 匹配规则 |
|------|---------|
| 依赖 | `node_modules/` |
| 构建产物 | `dist/`、`.next/`、`build/`、`coverage/`、`.turbo/` |
| 环境变量 | `.env`、`.env.*` |
| 系统文件 | `.DS_Store` |
| 独立 workspace 文件 | `pnpm-workspace.yaml`、`pnpm-lock.yaml` |

复制后将项目 `package.json` 的 `name` 字段替换为用户输入的名称。

## 依赖去重（方案 A：静态对比）

1. 读取根目录 `package.json` 的所有包名（`dependencies` + `devDependencies`）
2. 读取项目 `package.json` 的 `dependencies` 和 `devDependencies`
3. 按包名匹配，从项目 `package.json` 中移除根目录已有的包（不比较版本）
4. 将处理后的 `package.json` 写回项目目录

## 更新 pnpm-workspace.yaml

将新项目名称追加到 `pnpm-workspace.yaml` 的 `packages` 列表，若已存在则跳过。

## 安装

去重完成后，在 monorepo **根目录**执行 `pnpm install`。

## 技术方案

- 纯 Node.js 原生：`node:fs`、`node:path`、`node:child_process`（`execSync` 执行 pnpm install）
- 交互：`@inquirer/prompts`（`select`、`input`，已在 devDependencies）
- 入口：`scripts/new.mjs`（ESM）
