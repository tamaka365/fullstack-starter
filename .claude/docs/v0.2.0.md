# v0.2.0 — templatize 命令

## 目标

实现 `scripts/templatize.mjs`：将 `frontend/` 或 `backend/` 目录保存为 `templates/<name>/` 下的可复用模板。

## 交互流程

```
$ pnpm templatize

? 选择要模板化的目录
  ❯ frontend

# 若 templates/frontend/ 下已有模板，列出供选择；末尾附加"新建模板"选项
? 选择模板
  ❯ nextjs-basic        （更新）
    nextjs-advanced     （更新）
    + 新建模板

# 选择"新建模板"时，追加询问名称
? 模板名称: nextjs-pro

✓ 已保存为 templates/frontend/nextjs-basic
```

- 若 `frontend/` 和 `backend/` 均不存在 → 输出错误后退出（exit code 1）
- 若只存在一个 → 跳过第一步选择，直接使用存在的那个
- 目标路径：`templates/frontend/<name>/` 或 `templates/backend/<name>/`
- 若 `templates/<type>/` 尚无模板，跳过列表直接询问新建名称
- 更新已有模板时直接覆盖，无需二次确认

## 排除规则

复制时忽略以下内容：

| 类型 | 匹配规则 |
|------|---------|
| 依赖 | `node_modules/` |
| 构建产物 | `dist/`、`.next/`、`build/`、`coverage/`、`.turbo/` |
| 环境变量 | `.env`、`.env.*`（任意后缀） |
| 系统文件 | `.DS_Store` |

## 模板后处理

复制完成后，修改 `templates/<name>/package.json` 的 `name` 字段为模板名称。

## 技术方案

- 纯 Node.js 原生：`node:fs`（`fs.cpSync` + `filter` 选项）、`node:path`
- 交互：`@inquirer/prompts`（`select`、`input`、`confirm`，已在 devDependencies）
- 入口：`scripts/templatize.mjs`（ESM，`"type"` 不需要改，直接 `.mjs` 扩展名）
