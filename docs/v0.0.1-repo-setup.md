# v0.0.1 — 仓库管理配置

## 目标

初始化 git 仓库，建立 `dev` / `main` 双分支策略，通过 `pre-merge-commit` hook 确保合并到
`main` 时自动过滤开发专用内容，同时不影响将来 clone `main` 的项目正常使用。

---

## 分支策略

| 分支 | 用途 |
|------|------|
| `dev` | 日常开发，包含完整内容 |
| `main` | 干净的模板分支，将来新项目从此 clone |

日常在 `dev` 开发，功能完成后 `git merge dev`（在 `main` 上执行），hook 自动完成过滤。

---

## 各内容的归属

| 内容 | dev | main | clone main 的新项目 |
|------|-----|------|-------------------|
| `packages/` `templates/` `scripts/create.mjs` | ✅ | ✅ | ✅ |
| `docs/` | ✅ | ❌ | ❌ |
| `scripts/sync-main.mjs`（若有） | ✅ | ❌ | ❌ |
| `frontend/` `backend/`（测试产物） | gitignore | ❌ | ✅ 可正常提交 |
| `.githooks/` | ✅ | ✅ | ✅（继承，但不会误触发） |
| `.gitignore.main` | ✅ | ❌（已转为 .gitignore） | ❌ |

---

## .gitignore 两个版本

- `dev` 的 `.gitignore` → 包含 `frontend/` `backend/`（测试产物不提交）
- `main` 的 `.gitignore` → 不包含，让将来的项目自己管理

实现方式：在 `dev` 上维护两个文件：
- `.gitignore` — dev 分支使用
- `.gitignore.main` — hook 触发时复制为 main 的 `.gitignore`，同时作为哨兵文件

---

## Git Hook

### 激活方式

`package.json` 的 `prepare` 脚本在 `pnpm install` 时自动执行：

```json
"prepare": "git config core.hooksPath .githooks"
```

**注意**：clone 后必须先运行 `pnpm install` 才能激活 hook。

### `.githooks/pre-merge-commit`

触发时机：`git merge` 完成暂存、提交之前（Git 2.24+）。

执行���辑：

1. 检查当前分支是否为 `main`，不是则直接退出（不影响 dev 上的任何合并）
2. 检查暂存区是否存在 `.gitignore.main`（哨兵文件，只在 starter 的 dev 分支上存在）
3. 未检测到哨兵 → 直接退出（不影响将来项目自己的 dev → main 合并）
4. 检测到哨兵 → 执行过滤：
   - `git rm -r --cached docs/` + `rm -rf docs/`
   - 将 `.gitignore.main` 内容写入 `.gitignore`，删除 `.gitignore.main`
   - `git add .gitignore && git rm --cached .gitignore.main`
