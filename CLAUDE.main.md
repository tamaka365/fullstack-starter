# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Tooling

- Package manager: **pnpm** (required)
- Styling: **vanilla-extract** (`@vanilla-extract/css`)
- Node.js ≥ 20, pnpm ≥ 9

## Common Commands

```bash
# Scaffold frontend and backend from templates
pnpm run create

# Run all packages in watch mode
pnpm dev

# Type-check all workspaces
pnpm typecheck
```

## Architecture

pnpm monorepo:

### `packages/` — Shared Libraries

- **`@starter/ui`** — React component library with vanilla-extract. Exports TypeScript source directly, consumed via `transpilePackages` in Next.js (no pre-compilation needed).
- **`@starter/hooks`** — React custom hooks (`useLocalStorage`, `useDebounce`).
- **`@starter/utils`** — Utility functions (`cn`).

All packages extend `../../tsconfig.base.json`.

### `templates/` — Project Templates

- **`templates/nextjs`** — Next.js 15 App Router, self-contained tsconfig, references `@starter/*` via `workspace:*`.
- **`templates/backend`** — NestJS 11, self-contained tsconfig.

Templates have their own independent tsconfig (do not extend root).

### `frontend/` + `backend/`

Generated by `pnpm run create`. Pre-registered in `pnpm-workspace.yaml`.

## Adding a New Package

1. Create `packages/<name>/` with `package.json` (`name: @starter/<name>`) and `tsconfig.json` extending `../../tsconfig.base.json`
2. Set `"exports": { ".": "./src/index.ts" }`
3. Add to `transpilePackages` in `templates/nextjs/next.config.ts`
